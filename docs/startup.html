%rehash¶

01_rehashx.py

The IPython magic %rehash allows you to reload all of your startup files and also adds system commands to the namespace!

Insofar, I haven’t noticed any significant slowdown in startup time as a result of this, and it hugely eases utilizing IPython as a system shell.

Convenience Functions¶

IPython provides the functions IPython.Application.initialized() and IPython.Application.instance(). As a result, each script can easily implement the following as a check to ensure that the global IPython instance is running.

In [1]: if __name__ == "__main__":
   ...:    from IPython import Application.initialized
   ...:    import sys
   ...: 
  File "<ipython-input-1-cb854d5e1a2f>", line 2
    from IPython import Application.initialized
                                   ^
SyntaxError: invalid syntax

Unfortunately, from IPython import Application may be a feature of IPython7.3, which at the time of writing was released 3 days ago. If you don’t have this version of the module, you can alternatively run

In [2]: from IPython.core.application import Application

In [3]: Application.initialized()
Out[3]: False

In [4]: IPython.Application.initialized?
Object `IPython.Application.initialized` not found.

Here’s the help from IPython.application.instance.

In [5]: IPython.Application.instance?
Object `IPython.Application.instance` not found.

Examples¶

Create a singleton class using instance, and retrieve it:

>>> from traitlets.config.configurable import SingletonConfigurable
>>> class Foo(SingletonConfigurable): pass
>>> foo = Foo.instance()
>>> foo == Foo.instance()
True

Create a subclass that is retrieved using the base class instance:

>>> class Bar(SingletonConfigurable): pass
>>> class Bam(Bar): pass
>>> bam = Bam.instance()
>>> bam == Bar.instance()
True

Original¶

This is the IPython startup directory

.py and .ipy files in this directory will be run prior to any code or files specified via the exec_lines or exec_files configurables whenever you load this profile.

Files will be run in lexicographical order, so you can control the execution order of files with a prefix, e.g.:

00-first.py
50-middle.py
99-last.ipy
